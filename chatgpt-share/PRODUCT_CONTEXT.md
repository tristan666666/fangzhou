# 方洲AI 当前产品说明

## 一句话定位

这是一个面向跨境品牌的 `红人BD任务系统`。

重点不是做一个大而全的 AI 工作台，而是把红人 BD 的业务流程做成：

`先有任务 -> 再有线索 -> 再有会话 -> 最后有结果`

## 当前只聚焦一件事

第一阶段只做：

- 红人BD

先不展开：

- 内容与社媒
- 转化与放大

## 产品里的核心对象

当前这个系统所有页面都应该围绕这五个对象组织：

1. 品牌
2. 任务
3. 线索
4. 会话
5. 结果

## 页面结构

当前页面已经按三段来理解：

### 1. 创建任务页

负责定义：

- 市场
- 垂类
- 平台
- 通道
- 达人要求
- 合作方式
- 佣金底线
- 目标触达人数

并生成：

- 任务草案

### 2. 任务执行页

负责展示：

- 当前任务口径下的漏斗
- 线索明细表
- 每条线索的当前状态
- 每条线索的处理模式
- 最近动作
- 下一步动作

### 3. 会话处理页

负责展示：

- 当前选中线索的会话
- AI 给出的回复建议
- 用户采用某个建议后的会话推进

## 当前使用的命名

产品里现在已经尽量改成人话：

- 连接状态 -> 数据源状态 / 通道状态
- 加入任务流 -> 创建任务
- 预览 -> 任务草案
- 交付包 -> 执行方案
- 自动BD -> 自动触达
- 辅助BD -> AI辅助回复
- 人工BD -> 人工接管

## 当前 demo 的问题

虽然现在已经有结构了，但还存在这些明显问题：

1. 指标和漏斗口径仍然不够强
2. 线索表还是 demo 数据，不是真实任务驱动
3. 会话处理区与线索表的绑定还不够深
4. 视觉层级和可读性还需要继续调整
5. 任务列表虽然比之前更清楚了，但仍然不够像真实业务系统
